# React Practicum

Учебный проект "Основы реактивных веб-приложений" — интернет-магазин с каталогом товаров, корзиной, оформлением заказов и панелью администратора. Состоит из двух независимых частей: backend на Laravel (REST API) и frontend на React.

## Стек технологий

**Backend**
- PHP 8.4, Laravel 13
- JWT-аутентификация (`php-open-source-saver/jwt-auth`)
- SQLite в качестве базы данных

**Frontend**
- React 19, TypeScript
- Vite
- React Router
- React Hook Form + Yup для валидации форм
- Axios для запросов к API
- Sass (модули) для стилей

## Запуск проекта

### Backend

Из корня проекта:

```bash
docker compose up -d --build -V && sleep 10
```

Команда поднимает контейнер с Laravel-приложением, применяет миграции, наполняет базу тестовыми данными (сеятся один раз) и запускает сервер по адресу `http://localhost:8000`. Пауза `sleep 10` нужна, чтобы дать контейнеру время на инициализацию (создание `.env`, генерацию ключа, миграции и сиды) перед первым обращением к API.

### Frontend

```bash
cd frontend/
npm i
npm run dev
```

Frontend поднимется в режиме разработки на Vite и по умолчанию обращается к backend по адресу `http://localhost:8000`.

## Ролевая модель

В системе выделены три уровня доступа.

**Неавторизованный пользователь**
- просматривает каталог товаров и карточку отдельного товара;
- добавляет товары в корзину;
- может зарегистрироваться или войти в систему.

**Авторизованный пользователь**
- всё, что доступно неавторизованному пользователю;
- оформляет заказ из корзины;
- просматривает список своих заказов и карточку заказа;
- редактирует профиль: имя и пароль;
- выходит из системы.

**Администратор**
- всё, что доступно авторизованному пользователю;
- создаёт, редактирует и удаляет товары в каталоге;
- меняет статус любого заказа.

Признак администратора хранится на пользователе (`is_admin`) и передаётся в JWT-токене как кастомный claim. На backend доступ к административным операциям (управление товарами, смена статуса заказа) защищён middleware `admin`, которое проверяет этот флаг.

## Архитектура frontend (FSD)

Frontend построен по методологии Feature-Sliced Design. Слои располагаются в `frontend/src` в порядке убывания уровня абстракции — каждый слой может использовать только нижележащие:

- **app** — инициализация приложения: точка входа, роутинг (`app/routes`), защищённые маршруты (`ProtectedRoute`), общий layout (`app/layouts`).
- **pages** — страницы приложения, каждая собирает виджеты и фичи в готовый экран: главная (`home-page`), товар (`product-page`), корзина (`cart-page`), заказы (`orders-page`), профиль (`profile-page`), авторизация (`auth-page`), админ-панель (`admin-page`).
- **widgets** — крупные самостоятельные блоки интерфейса, составленные из нескольких сущностей и фич: шапка сайта (`header`), каталог товаров (`product-catalog`).
- **features** — пользовательские сценарии с побочными эффектами: вход и регистрация (`login`, `register`), добавление в корзину (`add-to-cart`), оформление заказа (`create-order`), смена статуса заказа (`update-order-status`), создание и удаление товара (`create-product`, `delete-product`), обновление имени и пароля (`update-name`, `update-password`).
- **entities** — бизнес-сущности со своей моделью и API-запросами: `user`, `product`, `order`, `cart`.
- **shared** — переиспользуемый код без привязки к бизнес-логике: настройка axios и хранение токена (`shared/api`), UI-кит (`Button`, `Input`, `Dropdown`, `Text`, `ToggleSwitch`, `Icon` и т. д. в `shared/ui`), общие стили и утилиты (`shared/lib`, `shared/styles`).

Каждый слайс (модуль внутри слоя) делится на сегменты `ui`, `model`, `api`, `lib` и открывает публичный интерфейс через `index.ts`, что скрывает внутреннюю реализацию от остальных слоёв.

## Сущности системы

- **User (пользователь)** — имя, email, пароль, признак администратора.
- **Product (товар)** — название, описание, цена.
- **Order (заказ)** — принадлежит пользователю, содержит комментарий и статус; статус меняется администратором.
- **OrderItem (позиция заказа)** — связывает заказ с товаром, хранит количество и цену на момент покупки.
- **Cart (корзина)** — хранится на стороне frontend (в `localStorage`), список товаров с количеством до момента оформления заказа.
